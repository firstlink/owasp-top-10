ASI01 · Agent Goal Hijack

Teaching Lens

How to Explain ASI01

  • The issue is not just bad output. The agent's objective changes.
  • Natural-language content is untrusted, even when it looks like data.
  • A legitimate tool can still cause harm if the goal or recipient changes.
  • Every scenario should end with the control lesson: validate intent before action.

Mitigation Mapping

What the Trainer Should Reinforce

Treat language as untrusted

Documents, emails, web pages, calendar invites, and tool outputs should all be treated as hostile until validated.

Validate intent at runtime

Before high-impact actions, compare current agent intent to the original task and pause on unexpected drift.

Reduce blast radius

Apply least privilege, scoped tools, approval steps, and human review for goal-changing or sensitive actions.

Log goal drift

Track goal state, tool-use patterns, and deviations so hijacked behavior becomes visible to operations teams.
