OWASP ASI01

Agent Goal Hijack

The workflow still looks correct, but the intent has already shifted.

Attacker-controlled content changes the agent's objective, task selection, or decision path.

The Mechanism

How the hijack happens

Untrusted content reaches the same reasoning context and changes the next decision.

Why It Matters

Why agentic systems are uniquely exposed

Autonomous execution

Actions can fire before anyone notices the drift.

Mixed-trust inputs

Trusted instructions and attacker content enter the same context.

Silent drift

The output still looks valid after the objective has shifted.

Attack Channels

Where the trigger can enter

Indirect prompt injection is common, but ASI01 is broader than one input channel.

Emails PDF documents Web pages Database records Tool responses Agent messages
Also included in ASI01

Deceptive tool output, malicious artifacts, forged agent messages, and poisoned external data.

Key insight

The tool can still work normally. The decision is what changes.
