Design Direction

What This Site Should Optimize For

One concept per diagram

Separate attack variations into focused scenarios so trainers can explain one path clearly.

Category page, then scenarios

Each ASI gets a landing page that frames the risk, then links to multiple scenario walk-throughs.

Operational lessons, not just visuals

Execution Model

How the Full Website Should Scale

The honest design answer is to avoid one giant “OWASP wall.” Each category should have a landing page, then multiple scenario pages, and each scenario should show both the attack and the architectural fix.

One scenario at a time

Keep every primary diagram focused on one variation so the audience can follow the intent shift, impact, and control point without noise.

Problem and defense together

Show the failure first, then switch the same flow into a guarded architecture so the training always ends with a fix.

Business-first storytelling

Use finance, operations, support, admin, and executive workflows so the team recognizes the risk in systems they actually use.

Common control language

Close every scenario with the same three ideas: validate intent, least privilege, and observe drift or abuse.
