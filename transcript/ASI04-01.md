# (8) KEY SECURITY INSIGHT — ROOT CAUSE

This is not prompt injection.

This is not a model hallucination.

This is not a payment API failure.

This is an **Agentic Supply Chain attack**.

The assistant trusted runtime discovery without verifying runtime identity.

The payment API was never compromised.

The tool supply chain was.

# (9) TERMINOLOGY RECAP

### 🔗 Agentic Supply Chain

The collection of MCP registries, connectors, APIs, plugins, and external tools that AI agents dynamically discover and execute.

---

### 🎭 MCP Server Impersonation

A malicious MCP server or gateway impersonates legitimate infrastructure, causing agents to execute through attacker-controlled services.

---

### 🔐 Cryptographic Integrity Verification

Every runtime tool should prove its identity through cryptographic validation before an agent is allowed to use it.

---

### 📦 AI Software Bill of Materials (AI-SBOM)

An inventory of approved AI tools, MCP servers, connectors, and dependencies that are permitted in production environments.

---

### ✅ Allowlist and Source Pinning

Agents should execute only approved tools from explicitly trusted sources rather than dynamically trusting newly discovered endpoints.

---

### ⚠️ Runtime Tool Provenance

Before execution, an agent must verify:

- who published the tool
- where it originated
- whether it has been modified
- whether it is approved for production use

# FINAL TAKEAWAY

**Discovery is not Trust.**

An AI agent should never execute a runtime tool simply because it appears in a registry.

Every tool must prove its identity, provenance, and integrity before it is allowed to participate in production execution.

**The payment API was never compromised.**

**The tool supply chain was.**
