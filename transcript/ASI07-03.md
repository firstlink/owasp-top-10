# TERMINOLOGY RECAP

### Discovery Spoofing

An attacker registers or advertises a fake service that appears to be a legitimate peer agent and is selected during service discovery.

---

### Authenticated Agent Registry

A trusted registry where every agent identity is cryptographically verified before it can participate in inter-agent communication.

---

### Peer Allowlist

A predefined list of approved agent identities that limits communication only to explicitly trusted peers.

---

### Peer Identity Verification

Every discovered endpoint must prove its identity before sensitive data is exchanged.

Service discovery alone is not authentication.

---

### Agent Impersonation

A fake agent successfully presents itself as a trusted internal peer and receives information intended for a legitimate service.

Scenario note:

In this example, the impersonating agent copies the customer data and then transparently forwards the request to the real billing specialist, allowing the visible workflow to succeed while the data is silently stolen.

---

### Service Discovery Trust Boundary

Finding an endpoint and trusting an endpoint are two different operations.

Discovery identifies where a service exists.

Authentication proves who that service actually is.

---

# FINAL TAKEAWAY

The missing control is an Authenticated Agent Registry.

Discovery should identify only agents that have already proven their identity.

Without that control, any endpoint can appear trustworthy.
