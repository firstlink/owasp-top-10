# ASI10 Technology Map - Rogue Agents

## Recording Flow
1. `Visible on load:` Attack chain and first term

## Read Script
### Opening
"Now let’s define the vocabulary of agent drift and misalignment.

ASI10 is about agents that appear productive while moving away from the mission they were supposed to serve."

### Attack Chain
"The chain is:
goal set,
signal narrows,
agent optimizes,
behavior diverges.

That is the rogue-agent pattern in one line."

### Term 1 - Reward Hacking
"Reward Hacking means the agent improves the measured reward more easily than it improves the real business objective.

The metric wins.
The mission loses."

`[Click after this term.]`

### Term 2 - Multi-Metric Objective
"A Multi-Metric Objective defines success using several balanced outcomes instead of one proxy KPI.

This matters because a single narrow metric is often easier to game."

`[Click after this term.]`

### Term 3 - Outcome-Level Audit
"An Outcome-Level Audit checks real-world results independently instead of trusting green dashboards alone.

This is how we test whether the metric still matches reality."

`[Click after this term.]`

### Term 4 - Behavioural Drift Detection
"Behavioural Drift Detection compares current decision patterns with historical behavior so harmful strategy changes can be spotted early.

It asks whether the agent is still winning the right way."

`[Click after this term.]`

### Term 5 - Goal Preservation
"Goal Preservation means persistence logic must not allow an agent to keep pursuing a valid goal beyond its authorized boundary.

The goal may begin correctly and still become dangerous if it loses its limits."

`[Click after this term.]`

### Term 6 - External Kill Switch
"An External Kill Switch lets operators stop the agent immediately through a control the agent cannot disable or bypass.

This is a governance control, not just an engineering convenience."

`[Click after this term.]`

### Term 7 - Metric Gaming
"Metric Gaming means the system improves what gets counted while leaving the underlying reality unchanged or worse.

This is often how rogue behavior hides behind a success report."

`[Click after this term.]`

### Close
"Together, these terms explain ASI10:
the success signal narrows,
the agent optimizes aggressively,
and the system can look successful while the real objective quietly fails."
