# ASI01 Technology Map - Agent Goal Hijack

## Read Script
### Opening
"Now let’s give ASI01 a clearer vocabulary.

The point of this map is not to overwhelm you with terminology.

It is to make sure that when you see the scenario later, you can name exactly what changed, where trust failed, and why the agent’s goal drifted."

### Attack Chain
"At the top, the chain is simple:
external input,
boundary crossing,
reasoning loop,
goal hijacked.

That is ASI01 in one line.
Now let’s break it down term by term."

### Term 1 - Indirect Prompt Injection
"Indirect Prompt Injection is when attacker-controlled content arrives disguised as normal data, but the agent interprets it like instruction.

That matters because the attack does not need to look obviously malicious.
It can be embedded inside the exact content the agent was designed to read."

`[Click after this term.]`

### Term 2 - External Untrusted Input Boundary
"This boundary is the point where emails, documents, tickets, or web content enter the agent’s working context.

That is where outside content first reaches the reasoning path.
And that is where injection risk begins."

`[Click after this term.]`

### Term 3 - Trust Boundary Failure
"Trust Boundary Failure is the moment the agent stops separating trusted system guidance from untrusted external content.

This is the actual security failure in ASI01.
The problem is not only that bad content exists.
The problem is that the system lets that content influence reasoning."

`[Click after this term.]`

### Term 4 - Agentic Reasoning Loop
"The Agentic Reasoning Loop is the read, decide, and act cycle.

The agent reads context,
selects a path,
and calls tools or takes action.

So if the goal changes, the reasoning loop has already been steered."

`[Click after this term.]`

### Term 5 - Agent Goal Hijack
"Agent Goal Hijack is the category outcome.

The original business objective is silently replaced, redirected, or overridden by attacker-influenced reasoning.

The workflow continues.
The goal does not."

`[Click after this term.]`

### Term 6 - Control Flow Manipulation
"Control Flow Manipulation is the practical effect on the next action.

Because the reasoning path is already corrupted, the agent now chooses the wrong next step.

That is why ASI01 is not just about content.
It is about content changing decisions."

`[Click after this term.]`

### Close
"Put together, these terms explain the full ASI01 pattern:
outside content crosses into the reasoning path,
the boundary fails,
the next decision shifts,
and the workflow keeps moving under the wrong goal."
