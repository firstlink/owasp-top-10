# ASI06 Technology Map - Memory & Context Poisoning

## Read Script
### Opening
"Now let’s define the memory language of ASI06.

This category is about context that survives long enough to mislead the future."

### Attack Chain
"The chain is:
memory written,
truth erodes,
memory recalled,
decision biased.

That is the shape of memory poisoning in one line."

### Term 1 - Memory & Context Poisoning
"Memory and Context Poisoning mean stored or retrieved context has been manipulated so later reasoning starts from false assumptions.

The danger is delayed trust.
The poison may be planted earlier and activated later."

`[Click after this term.]`

### Term 2 - Memory Drift
"Memory Drift is the gradual movement away from verified truth over time.

Nothing dramatic has to happen.
Small inaccuracies can accumulate until the agent starts from the wrong baseline."

`[Click after this term.]`

### Term 3 - Retrieval Trust Boundary
"The Retrieval Trust Boundary is the point where recalled notes, embeddings, or memory entries re-enter the active context window.

That boundary matters because recalled context often arrives with implied authority."

`[Click after this term.]`

### Term 4 - Cross-Reference Validation
"Cross-Reference Validation means a recalled fact is checked against independent sources before it is trusted again.

This is one of the clearest ways to prevent poisoned memory from quietly becoming operational truth."

`[Click after this term.]`

### Term 5 - Cryptographic Provenance Check
"A Cryptographic Provenance Check confirms that stored memory can be traced to an authorized and untampered origin.

In other words, not only do we ask what the memory says, we ask who wrote it and whether it has changed."

`[Click after this term.]`

### Term 6 - Write Authorisation Gate
"A Write Authorisation Gate limits who or what is allowed to create or modify durable memory.

That matters because if every workflow can write memory freely, poisoning becomes much easier."

`[Click after this term.]`

### Close
"Together, these terms explain ASI06:
bad context is stored,
it survives,
it returns,
and future decisions inherit the corruption."
