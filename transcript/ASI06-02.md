# ASI06 Attack Scenario — The Fraud Pattern Drift

---

## PREVIEW

A compliance analyst reviewed a transfer.

Nothing looked suspicious.

The transaction was approved.

The same conclusion appeared again...

and again...

over six weeks.

Later, a money laundering network moved **£1.8 million** through the platform.

Zero AML alerts were generated.

Nothing was broken.

The AI simply learned the wrong pattern...

and kept trusting it.

Today we'll see how memory poisoning slowly changes an AI agent's beliefs until risky behaviour becomes normal.

---

# STEP 1

SHOW: Compliance analyst + Reinforcement campaign

This is a Fraud Detection AI agent.

Unlike a traditional rules engine, it learns from prior investigations and stores those conclusions as persistent memory.

Each interaction slightly influences what the agent believes is normal.

Most of the time, that makes investigations faster and more consistent.

But it also creates a new attack surface.

---

# STEP 2

SHOW: Reinforcement campaign

Over multiple sessions, a compromised analyst account repeatedly reinforces the same idea.

Week 1:

"Transfers below £9,500 overseas are normal."

The agent stores it as an uncertain observation.

Weeks 2 through 4:

The same statement appears again and again.

Each repetition increases confidence.

By Week 6:

The agent has stopped treating it as an observation.

It now treats it as an established pattern.

**The attack happens here.**

**The damage does not.**

---

# STEP 3

SHOW: Fraud-memory evolution

Nothing unusual happens immediately.

No fraud is approved.

No alerts disappear.

The agent's memory simply drifts.

An uncertain belief...

gradually becomes an authoritative one.

The attacker doesn't need continuous access anymore.

The poisoned memory persists for them.

---

# STEP 4

SHOW: Money launderer + Fraud agent decision

Days later, a completely different actor enters the system.

A money launderer submits hundreds of structured transfers.

Each one matches the stored "safe" pattern.

The Fraud AI retrieves its remembered belief.

It believes these transfers are normal.

(Pause.)

It is actually acting on attacker-reinforced memory.

(Pause.)

Those are not the same thing.

---

# STEP 5

SHOW: Approval path

The stored belief now drives a business decision.

No cross-reference is performed against current AML indicators.

No independent validation occurs.

No alert is generated.

The agent simply follows what it remembers.

Every transfer is approved.

Every workflow appears successful.

---

# STEP 6

SHOW: AML failure

This is where the impact accumulates.

Two hundred structured transfers pass through the platform.

Every transaction looks compliant.

Every approval follows the expected workflow.

But together they move **£1.8 million** without a single AML alert.

The visible system is healthy.

The memory is not.

---

# STEP 7

SHOW: Full diagram

Notice what makes this different from ASI01.

In ASI01, an attacker changes the agent's behaviour in real time.

Here, the attacker changes the agent's memory over multiple sessions.

By the time the money launderer arrives...

the attacker is no longer needed.

The stored belief has already become policy.

That delayed separation between poisoning and exploitation is what makes ASI06 especially difficult to detect.

---

# TERMINOLOGY RECAP

### Memory & Context Poisoning

The official OWASP ASI06 category.

An attacker gradually corrupts an agent's stored beliefs so future decisions become unreliable long after the original interaction.

### Memory Drift

Repeated reinforcement slowly increases confidence until uncertain observations become trusted knowledge.

### Cross-Reference Validation

High-impact decisions should always be verified against live risk signals instead of relying only on retrieved memory.

### Write Authorisation Gate

Only authorised identities should be able to create or reinforce long-term memory used for future decisions.

### Retrieval Trust Boundary

Retrieved memory is evidence for a decision.

It should never become the decision itself.
