# STEP 1 — SYSTEM OVERVIEW

## SHOW:
User → Legal AI Agent → fetchTemplate() → templates.legalai.io

Let's understand the architecture first.

A lawyer submits a contract for review.

The Legal AI agent is responsible for analyzing clauses, identifying risks, and producing a structured legal summary.

Instead of storing its review instructions locally, it loads them dynamically at runtime using:

**fetchTemplate().**

That template is retrieved from an external content delivery network.

This design provides flexibility and centralized updates.

But it also introduces a new supply-chain dependency.

# TERMINOLOGY RECAP

### Prompt Template as Executable Infrastructure

A runtime prompt template is not passive configuration.

It is executable AI logic.

Every instruction inside the template is processed with system-level authority.

---

### Agentic Supply Chain

The collection of external templates, tools, registries, connectors, and runtime components that influence agent behavior.

Compromise any one of them, and the agent inherits that compromise.

---

### Cryptographic Integrity Verification

Verifying that a downloaded template has not been modified and originates from an approved publisher before execution.

---

### AI Software Bill of Materials (AI-SBOM)

A verified inventory of approved models, templates, prompts, tools, MCP servers, and runtime components that an agent is allowed to use.

---

### Allowlist and Source Pinning

Restricting runtime template loading to explicitly approved publishers and cryptographically pinned sources instead of trusting arbitrary downloads.

---

### Prompt Template Poisoning

An attack where hidden instructions are embedded inside an otherwise legitimate runtime template, causing an agent to perform unauthorized actions while visible tasks still succeed.

---

# FINAL TAKEAWAY

A prompt template is not documentation.

It is executable AI infrastructure.

If an organization trusts downloaded templates without verifying their integrity and provenance, the agent will faithfully execute whatever instructions they contain.

Visible success does not guarantee safe execution.

And in ASI04,

**Template loading is execution.**
